# FP8 engine variants

Each row below corresponds to a folder under `variants/<tag>/` containing:

- `engine.engine` — the actual TRT engine binary
- `engine.metadata.json` — TRT build metadata (version, GPU, ONNX SHA)
- `fp8_patch_manifest.json` — exact FP8 quant config and per-Linear log
- `config.json` — exact patch + build + listen commands to reproduce
- `metrics.json` — cos sim, speedup, MatMul counts
- `validation_results.json` — full benchmark JSON
- `audio/01_dance.wav` … `06_folk.wav` — listen-test renders (30 s @ seed 42)

The bf16 reference engine is archived at `variants/bf16/` for safekeeping
and its audio at `variants/bf16/audio/`. All FP8 variants are compared
against bf16 using identical seeds, prompts, and durations.

## Calibration

Two calibration generations exist:

- **cal1** (original): 8 prompts × 16 DiT calls × batch=4 = 64 samples.
  Backed up at `calibration.npz.v1` / `activation_absmax.json.v1`.
- **cal2** (current): 25 prompts × 200 DiT calls × batch=4 = 800
  steady-state samples. Generated by the same
  `scripts/collect_decoder_calibration.py` after extending the prompt
  list with vocal-forward, transient-extreme, low-freq-dominant,
  high-freq-dominant, dense-polyphony and sparse-content variants.

cal2 surfaced outliers that cal1 was under-sampling: layer 15
`mlp.down_proj` absmax jumped 4128 → 22272 (5.4x), layer 17 went 1416 →
15232 (10.8x). Bulk `p99.9` mostly held, so it's purely the extreme-tail
estimate that improved. Variants suffixed `_cal2` are rebuilds using
this calibration with no other config changes.

record so the failure mode is reproducible).

`W8A8 / W8A16` is the count of MatMul nodes whose weight initializer
gets quantized: `W8A8` MatMuls have both inputs on FP8 Q-DQ chains (TRT
picks FP8 GEMM); `W8A16` MatMuls have FP8 weight but the activation
stays bf16 (outlier-skip fallback). Both numbers exclude the 129
MatMuls that have non-initializer weight inputs (attention Q×K^T etc).

The same `--outlier-skip-ratio` produces different W8A16 counts under
cal1 vs cal2 because the ratios shifted (cal2 found larger absmaxes).
The configs are the same; the data they consume is different.

## Reading the metrics

- **cos sim**: latent-space cosine vs bf16 from the validation .npz.
  Cheap, but doesn't catch what the VAE does to the latent error.
- **text-CLAP**: cosine(CLAP_audio_emb, CLAP_text_emb) for the prompt.
  Absolute prompt adherence. n=6 prompts ⇒ std ~0.10-0.13 ⇒ weak signal
  unless gaps are large.
- **audio-CLAP vs bf16**: cosine between fp8 and bf16 audio embeddings.
  LAION CLAP uses non-deterministic chunking on >10s clips, so
  bf16-vs-itself varies 0.85-0.95 across runs of `score_variants.py`.
  That's the metric's noise floor; treat differences <0.05 as noise.
- **mel-L1 vs bf16**: log-mel L1 distance at 48 kHz. Deterministic
  spectral fidelity on the actual waveform. **Lowest variance of the
  three quality metrics, the cleanest signal.**
- **MR-STFT vs bf16**: multi-resolution STFT loss (3 windows). Tracks
  mel-L1 almost perfectly here — they agree on every ranking.

## Reproducing a variant

Each variant's `config.json` has the exact `patch`, `build`, and
`listen` commands. Or use the all-in-one builder:

```
python benchmarks-pr17/fp8_build_variant.py \
    --tag <tag> \
    --description "<text>" \
    --smoothquant-alpha <float> \
    --outlier-skip-ratio <float> \
    --percentile absmax
```

That patches the ONNX, builds the engine, runs validation + listen
test, and copies all artifacts into `variants/<tag>/` while appending
a row to this INDEX.md. Engines are never deleted — every variant
keeps its own folder with the engine binary preserved.

To regenerate calibration (cal2) and recompute activation absmax:

```
python scripts/collect_decoder_calibration.py --num-prompts 25 --max-calls 200
python scripts/collect_activation_absmax.py
```

To recompute the CLAP + mel-L1 + MR-STFT columns for all variants:

```
python benchmarks-pr17/score_variants.py
```

It writes `variants/scores.json` and prints the metrics table.

## Observations

- **cal2 dominates cal1 at every comparable speed point.** The
  speed/quality curve shifted outward across the board. Every
  cal2-built variant is the dominant choice at its speed tier.
- **Mel-L1 standard deviation collapsed 2-4x across cal2 variants.**
  cal1 mel-L1 std was 0.24-0.55 (some prompts were notably worse than
  others). cal2 std is 0.06-0.12 (consistent across content). The
  wider calibration not only improved mean quality but made quality
  much more predictable across diverse prompts. For production
  reliability this matters more than the mean improvement.
- **w8a8_skip3_cal2 is the new quality ceiling.** cos 0.9786 is the
  highest fp8 cos ever recorded on this graph; mel-L1 0.441 is the
  lowest. Speed is essentially bf16 (1.02x) so it's a "max quality fp8
  reference" rather than a production target.
- **The SmoothQuant family stays decisively beaten on spectral fidelity.**
  sq_a05_skip5_cal2 (the previously-untested combo) gets cos 0.9742 —
  competitive on cos — but mel-L1 1.14 and MR-STFT 1.35. The cos vs
  spectral disagreement persists: SQ's smoothing-then-per-output-channel
  weight quant moves the latent error in a way the VAE amplifies in
  waveform space, regardless of calibration quality.
- **Percentile mode (p99_9) is a confirmed dead end.** Even with cal2's
  proper outlier coverage and skip=5 protecting the worst layers,
  setting the activation scale by p99.9 catastrophically clips signal
  on layers that have meaningful values above their p99.9 threshold
  (cos crashed to 0.738). Per-tensor absmax is the only viable
  activation scale for this DiT.
- **Top recommendations:**
  - **Fastest fp8 with strong quality:** `w8a8_absmax_cal2` (1.52x, cos 0.9718, mel-L1 0.499). **Production default for speed.**
  - **Best speed-quality balance:** `w8a8_skip20_cal2` (1.40x, cos 0.9703, mel-L1 0.478). Beats both skip8 cal1 (slower + worse) and the SQ family.
  - **High-quality middle:** `w8a8_skip8_cal2` (1.27x, cos 0.9702, mel-L1 0.476). Tied on mel-L1 with skip20_cal2 but slower; pick this if cos matters more than speed.
  - **Highest spectral fidelity with real speedup:** `w8a8_skip5_cal2` (1.15x, cos 0.9735, mel-L1 0.483).
  - **bf16-grade reference fp8:** `w8a8_skip3_cal2` (1.02x, cos 0.9786, mel-L1 0.441). Highest quality fp8 we can produce.
